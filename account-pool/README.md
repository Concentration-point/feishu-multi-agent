# account-pool

本地账号资源管理层原型，用于给 Codex / OpenClaw / CPA 提供统一的账号池、状态管理、激活切换与后续扩展接口。

## 当前阶段

当前已按 **A + C 的折中路线** 起骨架：
- 先落本机可维护的账号池与调度层
- 预留后续自动化注册/导入接口
- 暂不内置高风险注册逻辑

这版重点是把“账号资源管理层”搭稳，而不是先把注册器写成炸药包。

## 目录

- `config/`：配置文件
- `data/`：SQLite 数据库与账号材料
- `scripts/`：日常操作脚本
- `runtime/`：当前激活账号的运行时文件
- `adapters/`：Codex / OpenClaw / CPA 适配层
- `core/`：核心逻辑

## 推荐工作流

1. 先准备/导入账号材料
2. 跑健康检查
3. 选择一个可用账号
4. 激活到 runtime
5. 由上层工具消费 runtime 中的当前账号
6. 失败时记录并轮换

## 当前已实现

- SQLite 初始化
- 账号状态枚举
- 账号导入脚本
- 账号选择脚本
- 账号激活脚本
- 全量健康检查框架（当前为占位检查）
- 运行时文件输出
