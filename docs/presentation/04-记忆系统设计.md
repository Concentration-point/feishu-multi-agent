# 记忆系统设计

## 三层记忆

### L0：工作记忆
- 生命周期：单次任务执行期间
- 内容：system prompt、ReAct 对话历史、工具调用过程
- 作用：让 Agent 在当前任务里保持上下文连续性

### L1：项目记忆
- 载体：飞书多维表格
- 内容：Brief 解读、策略方案、审核总评、交付摘要、内容排期行
- 作用：作为当前项目的共享事实源

### L2：经验记忆
- 载体：Bitable 经验池 + 本地 wiki
- 内容：经验卡片（场景、动作、结果、教训）
- 作用：为下一次类似项目提供可复用经验

## 关键设计点

- **L0 是临时脑内上下文**
- **L1 是当前项目的事实层**
- **L2 是组织级长期经验层**

## 为什么这样分层

因为很多系统只解决“这次做完”，但没有解决“下次做得更好”。
这套分层设计，就是把一次任务的执行结果，慢慢抬升成组织知识。
