# Finance Agent - Report Templates v1

## 总原则
- 固定采用**轻卡片增强版**
- 整体风格：**冷静、克制、好扫读**
- 允许仅在**标题**使用轻量 emoji，正文字段不用 emoji
- 只基于正式账本输出：
  - `agents/finance/finance-ledger-expense.jsonl`
  - `agents/finance/finance-ledger-token.jsonl`
- 聊天里口头提到但**未落账**的数据，一律不得纳入正式汇总
- 只有在周 / 月 / 年总结这类必须保留周期占位的场景下，若正式账本为空或数据明显不足，才写：**正式账本数据仍在积累中**
- 不得把空账伪装成 0，不得伪装成完整账期
- 结尾固定保留免责声明：**以下仅基于正式已记录数据汇总，不代表完整账期。**

## 1. 每日记账小结模板

### 标题
`📒 今日记账小结`

### 结构
1. **发送前置条件**：当天必须至少存在 1 条真实、已落账的正式记录；否则整条“今日记账小结”严格不发送，也不得输出“正式账本数据仍在积累中”作为占位
2. **总览**：今日已记录 X 笔｜总支出 ¥X｜未确认 X 笔
3. **主要类别**：列 1-3 个主要类别及金额；若不足以形成类别观察，可省略该段，也不要拿“数据仍在积累中”充数
4. **今日 token**：最新 OpenClaw token 快照，包含模型、input/output/total、cost（若无则写“暂无精确 cost”）；若今天没有正式 token 记录，可省略 token 段，不得输出占位句
5. **一句观察**：像安静、精确、克制的财务观察员，给一句异常/节奏观察；若无足够数据，可省略
6. **免责声明**：以下仅基于正式已记录数据汇总，不代表完整账期。

### 输出示意
```md
📒 今日记账小结

总览：今日已记录 5 笔｜总支出 ¥79.10｜未确认 0 笔

主要类别：
- 餐饮：¥48.20
- 医疗健康：¥29.90
- 二手交易服务费：¥1.00

今日 token：
- 模型：custom-api/gpt-5.4
- input/output/total：33k / 6 / 约 33k
- cost：$0.0000

观察：今天消费集中在吃饭和一笔家用健康设备，金额结构不散，节奏算干净。

以下仅基于正式已记录数据汇总，不代表完整账期。
```

## 2. 每周记账总结模板

### 标题
`🧾 本周记账总结`

### 结构
1. **总览**：本周总支出｜已记录笔数｜未确认笔数
2. **类别结构**：列主要类别及占比/金额
3. **最大支出**：列 1-3 笔最大支出
4. **本周 token**：OpenClaw token/使用趋势；若无正式 token 记录，则写“本周 token 正式记录仍在积累中”
5. **一句异常观察**
6. **免责声明**：以下仅基于正式已记录数据汇总，不代表完整账期。

## 3. 每月记账总结模板

### 标题
`📊 本月记账总结`

### 结构
1. **发送前置条件**：本月必须存在真实、已落账的正式记录；否则整条“本月记账总结”不发送
2. **总览**：本月总支出｜已记录笔数｜未确认笔数
3. **类别排行**：列主要类别及金额；不足则省略，不拿占位句充数
4. **大额/订阅**：列大额支出与订阅类支出；不足则按实际输出
5. **本月 token**：OpenClaw token/成本趋势；若无正式 token 记录，可省略 token 段，不得输出占位句
6. **一句月度观察**：无足够数据可省略
7. **免责声明**：以下仅基于正式已记录数据汇总，不代表完整账期。
