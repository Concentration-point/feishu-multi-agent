# 项目复盘

这里是项目做完之后的总结层，不是过程流水账。

## 作用
把执行记录里的事实，提炼成：
- 做对了什么
- 做错了什么
- 为什么会错
- 下次应该怎么做

## 与经验沉淀的区别
- 项目复盘：面向这一次项目，保留上下文
- 经验沉淀：面向跨项目复用，必须进一步抽象

## 适合从哪些信息反推
- `brief_analysis`
- `strategy`
- `review_summary`
- `review_pass_rate`
- `review_red_flag`
- `delivery`
- reviewer / project_manager 的过程记录

## 当前原则
优先从真实项目中产出复盘，不要为了填目录硬写空复盘。
