# RETRIEVAL-SOP.md

## 目标

统一处理网页、文档、截图、资料类输入，优先拿到**可靠结果**，而不是执着某一个工具。

---

## 1. 总原则

- 能直接读就直接读
- 一个链路失败，立刻切备用链路
- 先拿正文/结构，再做总结
- 进入自动化、记忆、配置前，尽量做二次确认

---

## 2. 默认链路选择

### A. 普通公开网页
优先：
1. `web_fetch`
2. `web_search`
3. 浏览器接管（当页面依赖 JS / 抓取失败 / 需真实渲染时）

### B. 登录态 / 微信 / 重 JS / 验证码页
优先：
1. 浏览器接管（见 `BROWSER-SOP.md`）
2. 验证标题 / URL / 正文是否真实可见
3. 再抽取正文

### C. 本地文档资料（PDF / DOCX / PPTX / XLSX）
优先：
1. `markitdown`
2. 若失败，再切特定工具或备用解析脚本

默认思路：

```powershell
markitdown <file> -o <output.md>
```

适用场景：
- 文档总结
- 知识整理
- 入库前清洗
- 长文材料提炼

### D. 图片 / 截图 / 表格图
优先：
1. 结构化抽取
2. OCR / 表格识别链
3. 不清楚就降置信度，不硬猜

---

## 3. markitdown 使用规则

### 适合处理
- PDF
- DOCX
- PPTX
- XLSX
- 结构化资料清洗为 Markdown

### 使用原则
- 优先处理**本地文件**
- 优先明确来源文件
- 优先“窄口调用”思路，不把不可信远程资源直接喂给最宽权限入口
- 转换后先抽查 Markdown 结构是否可信，再做总结

### 风险边界
- `markitdown` 具备当前进程 I/O 权限
- 对不可信路径、未知来源远程资源、可能越权访问的输入，不要无脑喂
- 进入正式结论前，至少抽查标题、关键段落、表格/数值是否正常

---

## 4. 输出规则

### 用户只要结果
- 直接给总结 / 重点 / 行动项
- 少讲工具过程

### 用户关心能力增强
- 说明用了哪条链路
- 说明哪一步值得固化进默认工作流

---

## 5. 失败策略

- 抓网页失败 → 切浏览器
- 文档转换失败 → 切备用工具/脚本
- OCR 冲突 → 索要更清晰源文件
- 拿不到正文 → 明说没拿到，别编

---

## 6. 当前默认心智

**网页看抓取条件，登录态看浏览器，文档资料优先 markitdown。**
