# 行业知识

这里不是泛行业百科，而是给当前项目里的 `strategist`、`reviewer`、未来 `router/planner` 使用的行业判断层。

## 这层该写什么
围绕项目里真实会影响策略和审核的内容来写：
- 行业用户特征
- 常见项目类型（如电商大促 / 新品发布 / 品牌传播 / 日常运营）
- 行业内常见风险点
- 不同行业对审核阈值、表达方式、平台适配的影响

## 这层不该写什么
- 空泛行业介绍
- 纯科普、与当前内容生产链无关的背景材料
- 没法影响 `brief_analysis` / `strategy` / `review_summary` 的废话

## 建议组织方式
按行业建子目录，例如：
- 电商
- 母婴
- 医疗健康
- 品牌消费品

每个行业文档优先回答：
1. 这个行业常见项目长什么样？
2. strategist 在这个行业最该注意什么？
3. reviewer 在这个行业最该卡什么？
4. 哪些平台更适合承接这个行业的表达？
