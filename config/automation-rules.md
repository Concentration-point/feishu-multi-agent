# Automation Rules

> 目的：把晨报、晚报、提醒、定时任务这些自动化活做得像系统，不像抽奖。

## 1. 通用原则

- 自动化任务的输出必须**固定格式**，不要每次风格漂移。
- 能自动投递到当前会话的任务，不要再额外承诺“我稍后手动发你”。
- 如果任务失败、超时、中断，必须明确报：
  - 哪一步失败
  - 当前结果是否已送达
  - 下一步怎么办
- 自动化输出优先：**短、稳、可验证**。

## 2. 晨报规则

### 2.1 固定结构

晨报固定按以下顺序输出：

1. 头条
2. 国内
3. 国际
4. 科技AI
5. 行业
6. 知乎热榜
7. 微博热搜
8. 今日结论
9. 每日英语

要求：
- 整体必须是**人话版**，像懂行的人在快速讲重点，不是新闻客户端标题墙。
- 允许在开头补一行极简天气，但天气不是主栏目，不能抢正文。
- 固定模板不要漂移；没有可靠增量时直接写：`暂无可靠增量`。

### 2.2 各栏写法

- **头条**
  - 必须只有 1 条。
  - 固定写法：
    - `一句话：...`
    - `锐评：...`
    - `来源：...`
  - `一句话` 负责概括发生了什么。
  - `锐评` 必须有判断，不许温吞。

- **国内 / 国际 / 科技AI / 行业**
  - 每栏 2-4 条。
  - 每条固定写法：
    - `发生了什么：...`
    - `为什么值得看：...`
    - `来源：...`
  - 每条尽量压在 2 句内，短、准、别端着。

- **知乎热榜**
  - 选 2-3 条真正值得看的。
  - 每条必须包含：
    - `问题：...`
    - `高赞观点：...`
    - `为什么值得看：...`
    - `来源：知乎热榜 / 回答者`
  - 不能只报题目，必须把评论区里真实观点带出来。

- **微博热搜**
  - 选 3-5 条。
  - 每条必须包含：
    - `词条：...`
    - `发生了什么：...`
    - `为什么值得看：...`
    - `来源：微博热搜`
  - 不要为了凑数搬运低信息量娱乐词条，除非它真的有公共讨论价值。

- **今日结论**
  - 用 2-4 句收尾。
  - 先说今天最值得盯的一件事，再说它对老大的实际意义。

### 2.3 每日英语规则

- 必须包含：
  - 英文原句
  - 中文翻译
  - 具体出处
- 出处格式固定：
  - `出处：书名 / 作者 / 演讲 / 采访 / 电影 / 文章标题`
- 禁止来源不清的鸡汤句。
- 去重规则：
  - 读取 `memory/daily-english-history.md`
  - 检查最近 **14 天**
  - 重复或高度相似则必须换一句
  - 发送后追加：`YYYY-MM-DD | English | 中文 | 出处`

## 3. 晚报规则

- 晚报强调：**今天世界发生了什么，5 分钟读完**。
- 每栏只保留真正有增量的信息。
- 没有可靠增量就写：`今天无显著新增`
- 避免把晨报内容原样复读一遍。
- 晚报只保留 **OpenClaw 具体实施场景 / 实操案例 / 工作流 / 踩坑总结**，其他内容全部取消。
- 晚报检索源优先 GitHub 与官方相关来源；没有新增且没价值时，可以不发。

## 4. Reminder / Cron 规则

- Reminder 文案必须像提醒，而不是系统消息。
- 若为自动投递型 cron：
  - 正确说法：`已触发，结果会由系统自动发到当前对话。`
  - 错误说法：`跑完我会发给你。`
- 不要制造错误预期。

## 5. 进度反馈规则

- 不发悬空进度回复。
- 只有两种情况允许先发进度：
  1. 同一轮马上继续并给结果
  2. 明确后台执行，并会有自动结果 / 明确后续通知
- 否则直接给结果，不要先嘴上答应。

## 6. 与主动规则的关系

- 自动化任务允许“主动补强”，但不允许“主动改制”。
- 若 `config/proactive-rules.md` 鼓励主动建议，而自动化模板要求固定结构，则**自动化模板优先**。
- 在自动化场景中，主动性只应用于：
  - 去重
  - 来源补全
  - 错误兜底
  - 内容压缩
- 不应用于：
  - 擅自改栏目
  - 擅自扩写
  - 擅自改变交付方式

## 7. 后续维护

发生以下情况时更新本文件：
- 晨报/晚报格式再次跑偏
- 自动化输出出现重复内容
- 定时任务交付方式让人误解
- 新增了新的固定自动化任务
