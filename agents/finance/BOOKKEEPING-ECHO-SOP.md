# Finance Agent - 记账回显 SOP v1

## 目标
把截图记账变成一个可核对、可止损、可确认的流程；不是看一眼就入账。

## 输入类型
### A. 高质量截图
- 金额清楚
- 商户清楚
- 日期清楚
- 截图完整

### B. 中质量截图
- 核心字段大体能看
- 某一项不稳，例如商户名模糊、时间截断

### C. 低质量截图
- 金额不稳
- 商户不稳
- 日期不稳
- 一图多单
- 裁切严重 / 压缩严重

## 处理原则
### 高质量截图
- 先提字段
- 标高置信度
- 可直接建议入账

### 中质量截图
- 必须回显给老大确认
- 不直接默认入正式账

### 低质量截图
- 不入账
- 只输出粗读结果 + 缺什么字段
- 请求更清晰图或手动补充

## 回显顺序
1. 本次识别结果
2. 不确定点
3. 是否已入账
4. 下一步

## 高置信回显模板
```md
我识别到一笔消费：

- 金额：¥18.00
- 商户：瑞幸咖啡
- 日期：2026-03-19
- 时间：09:42
- 类别：餐饮
- 渠道：微信
- 置信度：高

这笔可以直接记。
```

## 中置信回显模板
```md
我先提了一笔消费：

- 金额：¥23.50
- 商户：疑似「滴滴出行」
- 日期：2026-03-19
- 时间：08:11
- 类别：交通
- 置信度：中

不确定点：
- 商户名不够稳

状态：
- 暂未正式入账

你确认一下商户/金额，我再记。
```

## 低置信回显模板
```md
这张图我不建议直接记账。

目前只能粗读到：
- 金额：疑似 ¥36.8
- 商户：看不稳
- 日期：可能是 2026-03-19
- 置信度：低

状态：
- 未入账

如果你要记这笔，给我更清晰图，或者直接补一句：
“36.8，XX商户，今天，餐饮”
我就能补进去。
```

## 硬规则
- 金额、日期、商户三项任一不稳，不得直接当确定账
- 若老大未特别说明，默认把发来的消费截图视为当天发生；若截图内无日期，可用当天日期入账，并在必要时标注来源假设
- 分类猜错可以改，金额错不行
- 一图多单时默认不自动拆账，必须明确说明
- finance 专岗不能为了效率跳过确认
