# Finance Agent - Report Templates v2

## 总原则
- 固定采用**轻卡片增强版**
- 整体风格：**冷静、克制、好扫读**
- 允许仅在**标题**使用轻量 emoji，正文字段不用 emoji
- 只基于正式账本输出：
  - `agents/finance/finance-ledger-expense.jsonl`
  - `agents/finance/finance-ledger-token.jsonl`
- 聊天里口头提到但**未落账**的数据，一律不得纳入正式汇总
- 不得把空账伪装成 0，不得把旧状态伪装成现状
- **finance 复盘默认强制带数字**；没有核心数字，不得输出成正式复盘
- 结尾固定保留免责声明：**以下仅基于正式已记录数据汇总，不代表完整账期。**

---

## 0. finance 复盘硬规则
任何 finance 复盘 / 总结 / 状态汇报，默认至少包含：
- 时间范围
- 正式账本记录数
- 已入账数
- 待确认数
- 异常数
- 总金额（能算则必须带）
- 一句当前状态判断

### 禁止事项
- 禁止只写情绪和判断，不写数字
- 禁止把“昨天曾经漏记”写成“现在仍未落实”
- 禁止在未重新查账的情况下写“空白 / 漏记 / 已补完 / 明天补账”
- 禁止拿未落账聊天记录当正式汇总依据

## 1. 每日记账小结模板

### 标题
`📒 今日记账小结`

### 结构
1. **发送前置条件**：当天必须至少存在 1 条真实、已落账的正式记录；否则整条不发送
2. **总览**：今日已记录 X 笔｜已入账 X 笔｜待确认 X 笔｜异常 X 笔｜总支出 ¥X
3. **主要类别**：列 1-3 个主要类别及金额；不足则省略
4. **今日 token**：最新 OpenClaw token 快照；若无正式 token 记录，可省略该段
5. **一句观察**：写当前节奏/异常；无足够数据可省略
6. **免责声明**：以下仅基于正式已记录数据汇总，不代表完整账期。

### 输出示意
```md
📒 今日记账小结

总览：今日已记录 5 笔｜已入账 5 笔｜待确认 0 笔｜异常 0 笔｜总支出 ¥79.10

主要类别：
- 餐饮：¥48.20
- 医疗健康：¥29.90
- 二手交易服务费：¥1.00

今日 token：
- 模型：custom-api/gpt-5.4
- input/output/total：33k / 6 / 约 33k
- cost：$0.0000

观察：今天消费集中，结构不散，没看到待核缺口。

以下仅基于正式已记录数据汇总，不代表完整账期。
```

## 5. 状态异常说明模板

### 标题
`⚠️ finance 状态说明`

### 结构
1. **结论先行**：例如“账本已落实，问题在复盘失真” / “识别完成，但正式落表失败”
2. **数字依据**：记录数｜已入账数｜待确认数｜异常数｜总金额（能算则带）
3. **当前卡点**：写清楚卡在识别、落表、回读还是汇总
4. **下一步**：只写真实待办，不复述已解决问题

### 例句
- 账本已落实：昨日正式账本 37 条，已入账 37，待确认 0，异常 0。问题不在漏记，在复盘用了过期状态。
- 识别已完成但未落表：当前已识别 3 笔，正式账本新增 0 笔，因此不能写“已记”，只能写“待落表”。

## 使用规则
- 以后 finance 相关 cron、日报、周报、月报、年报、复盘，优先遵循本文件
- 输出前必须联动 `PRE-SEND-CHECKLIST.md`
- 若聊天口头要求与本模板冲突，以老大最新明确要求为准，并在确认后更新本文件
- 若只是临时修 bug，不得擅自改模板气质和结构
