# 亮点与答辩问答

## 项目亮点

### 1. 不是单 Agent，而是角色化协作
很多项目只是一个模型接一个 prompt。这个项目把真实团队里的角色关系拆开了。

### 2. 不是一次性输出，而是全链路执行
从 Brief 解读、策略拆解、文案生成、审核、交付，到最终知识沉淀，是一条完整业务链。

### 3. 不是纯文本对话，而是和飞书业务系统深度耦合
项目直接落在飞书多维表格、IM、知识空间这些企业常用载体上，更接近真实工作场景。

### 4. 带自进化闭环
经验会进入长期记忆，在后续项目中被检索复用。这是“越跑越聪明”的关键。

### 5. 跨项目数据分析（对应参赛模块 4）
数据分析师作为独立 Agent，从三张多维表格拉取全量业务数据，生成运营周报、数据洞察、决策建议三种报告，通过飞书群聊推送。不只是“跑完流水线”，而是能基于业务数据做出分析和决策。

## 常见问题

### Q1：为什么不用现成 Agent 框架？
A：因为项目目标之一就是证明，基于 function calling + 配置驱动，也能搭出清晰、可控、可解释的 Agent 系统。

### Q2：为什么知识库采用本地优先？
A：因为 Agent 检索更快、更稳定，飞书知识空间更适合作为人的查看界面，不适合做高频读取源。

### Q3：如何避免低质量经验污染经验池？
A：通过 Hook 蒸馏、置信度打分、阈值过滤、去重合并四层机制做控制。

### Q4：为什么要设计审核返工？
A：因为真实内容生产不是一次生成就结束，审核反馈和迭代是业务流程里不可省略的一环。

### Q5：数据分析师和流水线是什么关系？
A：数据分析师独立于项目流水线，不处理单个项目，而是跨项目汇总三张多维表格的业务数据，生成统计报告和决策建议。可通过 CLI、Dashboard API 或定时任务触发。

### Q6：当前还有什么不足？
A：目前还有历史乱码、测试环境依赖不完整、答辩素材图形化不够，这是工程收尾问题，不是主链缺失。
