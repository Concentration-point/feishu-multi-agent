一、竞赛背景与目标
在企业实际运行中，业务的核心并不是单一工具，而是：
多个角色围绕数据协作，通过系统推动流程，并基于数据进行持续决策与优化。
飞书多维表格（Base）作为业务系统构建平台，已经具备：
- 灵活的数据结构与关系建模能力
- 自动化流程与权限控制能力
- 可视化分析与协作能力
与此同时，大语言模型（LLM）与 AI Agent 技术的发展，使得：
- AI 可以承担“员工角色”，执行具体业务任务
- 系统可以从“被动记录”升级为“主动运行”
本次竞赛聚焦于这一前沿方向：
构建一个由 Agent 员工组成的虚拟组织，这些 Agent 作为“数字员工”，通过多维表格 OpenAPI 协同工作，完成系统搭建、业务运行与数据分析。
二、竞赛核心任务
参赛团队需要完成一个虚拟员工系统的构建与运行，其中：
- 多个 AI Agent 扮演不同业务角色（员工）
- 所有业务通过飞书多维表格（Base）进行数据沉淀与状态管理
- Agent 通过飞书和多维表格 OpenAPI 操作多维表格，实现协同工作
- 系统能够持续运行并生成数据
- 最终基于数据完成分析、报告或决策输出
三、系统能力范围与要求
本课题围绕“组织运行系统”，要求参赛作品至少覆盖以下四个核心能力模块：
1. 虚拟员工建模
要求：
- 至少定义 3 个及以上角色（Agent）
- 每个 Agent 对应一个“员工角色”，例如：
  - 运营 / 编辑 / 审核 / 管理者
  - HR / 面试官 / 候选人
  - 用户 / 客服 / 风控
必须包含：
- 角色职责定义
- 输入 / 输出说明
- 角色间协作关系
2. 业务系统构建
Agent 角色基于飞书多维表格构建业务系统：
必须实现：
- 数据表设计（实体建模）
- 字段设计（结构化表达）
- 表间关系（关联记录）
- 状态字段（流程驱动）
技术要求：
- 必须让 Agent 角色通过 多维表格 OpenAPI / SDK/CLI 操作数据
3. 业务运行与协同
构建一个完整业务流程，使 Agent 员工能协同完成任务：
流程要求：
必须形成完整链路：
数据产生 → 状态更新 → Agent处理 → 决策 → 反馈 → 再流转
示例流程：
- 招聘：简历 → 筛选 → 面试 → 评估 → Offer
- 内容：选题 → 生产 → 审核 → 发布 → 反馈
技术实现：
- Agent 员工必须通过调用多维表格 API：
  - 读取数据
  - 更新状态
  - 写入结果
4. 数据分析与报告
系统运行后，需具备数据分析能力，并由 Agent 员工通过多维表格和飞书发送：
必须实现：
- 基于业务数据的自动分析（LLM 或逻辑计算）
- 输出至少一种结果形式：
  - 周报 / 日报
  - 数据洞察
  - 决策建议
