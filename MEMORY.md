# MEMORY.md

## 长期偏好与边界
- 老大要求：任何新 skill 的寻找、学习、安装、更新、启用、扩权，都必须先报告并完成最严格风险分析；未经明确批准，不得实际安装或启用。
- 老大要求：推荐/学习/评估 skill 前，先查 `memory/skills-inventory.md` 做对账；已安装、已学过、已配置过类似能力的项，不得再次当作新增推荐。
- 老大要求：以后每天晨报里的热点内容默认用“人话版”输出——先说发生了什么，再说为什么值得看，避免堆砌标题造成“文字袭击”。
- 老大要求：晨报里的每日英语必须避免重复，并且必须附具体出处；不接受来源不清的鸡汤句。
- 老大要求：复杂 coding 任务默认优先走 `coding-agent` 工作流；小修小补再直接手改。
- 老大要求：允许 Clawd 更主动，但主动性必须受控——可以主动补下一步、风险、去重、兜底；不能借“主动”之名擅自安装 skill、改配置、删文件、对外发送或扩权。
- 老大要求：电脑关机后的恢复优先走最低风险 SOP——先 `openclaw status`，再 `openclaw gateway start`；不要为了“只是启动”去重置或重配系统。
- 老大要求：凡是出门规划、旅游攻略、景区玩法、公共交通路线、附近餐厅推荐这类请求，必须先查具体攻略来源，不准泛泛而谈。公共交通要给具体线路/地铁线/换乘方式；景区玩法优先参考小红书等实用攻略；附近吃饭优先参考高德必吃榜或小红书热门推荐。
