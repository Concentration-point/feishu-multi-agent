# Finance Pre-Send Checklist

任何 finance 相关的对外输出（记账回执、复盘、小结、日报、周报、月报、异常说明）发送前，必须过这张单子。

## 1. 数据源检查
- [ ] 本次结论基于正式账本 / 正式回读结果，不是只基于聊天过程
- [ ] 若引用“今天/昨天/本周/本月”状态，时间范围已明确
- [ ] 已确认本次使用的是真正生效的账本源（当前主账本为飞书多维表格，不默认等于本地 JSONL）

## 2. 数字检查
- [ ] 已提供记录数
- [ ] 已提供已入账数
- [ ] 已提供待确认数
- [ ] 已提供异常数
- [ ] 能算总金额时已提供总金额

## 3. 状态词检查
如果文案里出现以下任一词，必须触发额外核验：
- 已记
- 已入账
- 已落实
- 漏记
- 空白
- 未落实
- 明天补账
- 已补完

触发后必须确认：
- [ ] 已重新查询正式账本 / 回读结果
- [ ] 当前文案反映的是**现在**，不是旧状态

## 4. 闭环检查
- [ ] 若声称“已记/已入账”，则已确认写入成功
- [ ] 若声称“已记/已入账”，则已确认回读成功
- [ ] 若任一步失败，文案已改为“待落表/待回读/待核对/写入失败”

## 5. 复盘检查
- [ ] 复盘写的是当前结论，不是过期剧情
- [ ] 若问题已处理，已明确写“已处理”，并把剩余问题归因到流程/文案/检查机制
- [ ] 没有把“已解决问题”继续冒充待办

## 一句话
**没过单子，就别发。**
