# Subagent Trigger Rules

目标：复杂任务默认更早切子代理，主对话保持清爽。

## 默认触发条件
满足任意一项，优先考虑子代理：
- 需要修改 3 个及以上文件
- 需要构建 / 测试 / 调试超过 1 轮
- 需要长时间搜索、审计、整理大量材料
- 需要隔离上下文，避免主对话被施工细节污染
- 需要并行探索多个方案

## 主代理职责
- 明确目标
- 拆解任务
- 分配给子代理
- 汇总结果
- 做最终验收与对外输出

## 子代理职责
- 在限定范围内执行
- 产出修改、结论或中间结果
- 明确报告完成 / 失败 / 阻塞点

## 默认不切子代理的场景
- 单文件小修
- 一次性信息查询
- 简单文本润色
- 几分钟内可收口的小任务

## 输出规则
- 主对话不回灌大量施工日志
- 子代理结果必须先收敛成结论，再发给用户
