# Demo 讲解词

## 开场

我们现在演示的是：用户只在飞书里新建一条 Brief，系统就自动组建一个虚拟内容团队，完成从策略到交付的全链路工作。

## 演示步骤

### 第一步：新建 Brief
看这里，我在飞书多维表格里新增一条项目记录。
这个动作要么通过 Webhook 自动触发，要么也可以用 CLI 直接跑。

### 第二步：系统启动流水线
此时 Orchestrator 会按顺序启动 5 个角色：
客户经理、策略师、文案、审核、项目经理。

### 第三步：观察主表字段逐步填充
你会看到 Brief 解读、策略方案、审核总评、交付摘要这些字段，不是一次性凭空出现，而是按角色分工逐步被写回。

### 第四步：观察内容排期表
策略师先拆出内容项，文案补成稿，审核给反馈，项目经理安排发布日期。
这说明系统处理的不是一段孤立文本，而是一套内容生产任务。 

### 第五步：观察知识沉淀
项目完成后，经验会被蒸馏进经验池与本地 Wiki，再同步到飞书知识空间。
这意味着下一次类似项目不需要从零开始。 

### 第六步：数据分析报告

流水线结束后，我们再看模块四——数据分析与报告。
执行 `python main.py report --type weekly`，数据分析师会从多维表格的三张表（项目主表、内容排期表、经验池）聚合业务数据，
自动生成一份运营周报，包含项目完成率、审核通过率、平台分布、经验池健康度等指标，
并附上 2-3 条可操作的决策建议，最后通过飞书群聊推送给团队。

这个角色不参与流水线，而是站在全局视角做跨项目分析。

## 收尾句

这套系统的重点不只是"能写文案"——五个角色走完内容生产全链路，第六个角色站在全局视角做数据分析，经验沉淀让系统越跑越聪明。协作逻辑、数据决策和组织沉淀，一起做成了自动化。
