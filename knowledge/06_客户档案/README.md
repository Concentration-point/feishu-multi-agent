# 客户档案

这里服务的是项目主表里的 `client_name`、`brand_tone`，以及 account_manager / strategist / project_manager 的上下文读取。

## 作用
客户档案不是展示页，而是跨项目复用的客户事实层。

同一个客户可能会有多个项目：
- 电商大促
- 新品发布
- 品牌传播
- 日常运营

客户档案回答的是：**这个客户长期是谁、风格怎样、禁区在哪。**

## 建议包含
- 客户名称（对应项目主表 `client_name`）
- 所属行业
- 品牌定位
- 品牌调性（对应 `brand_tone`）
- 核心产品 / 服务
- 目标用户
- 常见审核偏好
- 风险禁区
- 历史合作项目索引

## 和项目档案的区别
- 客户档案：长期稳定，跨项目有效
- 项目档案：一次执行，跟具体 brief 强绑定

## 当前原则
没有真实客户信息前，不要硬写假档案。
先留空位，后续从正式项目中反推填写。
