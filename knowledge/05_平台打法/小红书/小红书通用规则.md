# 小红书通用规则

## 平台核心逻辑

小红书产出物是**图文笔记**（标题 + 封面描述 + 正文 + 标签），核心形态是"种草"——用真实体验驱动收藏和搜索。小红书同时也是强搜索平台，笔记需要兼顾推荐流量和搜索流量。

算法权重：点击率（封面+标题）> 互动率（收藏 > 评论 > 点赞）> 完读率。收藏率是小红书最核心的指标，收藏率高 = 内容有长期参考价值。

## 笔记字数控制
- 轻种草/好物分享：300-500 字
- 深度测评/攻略：500-800 字
- 上限 1000 字，超过阅读完成率会下降

## 笔记结构公式（所有行业通用）

| 段落 | 任务 | 要点 |
|------|------|------|
| 标题 | 决定点击率 | 20 字以内，含关键词，用数字/痛点/人群锁定 |
| 封面 | 决定曝光 | 高清、信息量大、有文字标注重点 |
| 开头 2 行 | 决定是否展开 | 痛点共鸣 / 结果先行 / 身份代入 |
| 正文主体 | 交付价值 | 分段清晰、emoji 分隔、重点加粗、图文结合 |
| 结尾 | 引导互动 | 提问/求评论/引导收藏 |
| 标签 | 搜索流量 | 3-5 个话题标签，含 1 个大词 + 2-3 个细分词 |

## 标题公式（任选其一）
1. **人群+痛点+结果**：「油皮姐妹看过来，这款粉底液持妆 12 小时」
2. **数字+干货**：「5 个让厨房收纳翻倍的神器」
3. **反常识/揭秘**：「求求你别再这样洗脸了」
4. **场景+方案**：「上班族 15 分钟快手早餐，一周不重样」
5. **情绪+事件**：「后悔没早知道！这个方法太绝了」
6. **对比/效果**：「用了 1 个月，素颜出门被问是不是化了妆」

## 标题红线
- ❌ 标题里放品牌名（容易被判广告限流）
- ❌ 绝对化用语（最好/第一/最便宜 → 改为"性价比高""闭眼入"）
- ❌ 标题太长超过 20 字
- ❌ 标题和内容不符（标题党会被降权）

## 正文风格
- ✅ 第一人称分享体（「我」「亲测」「自用」）
- ✅ 口语化、闺蜜聊天感
- ✅ 每段 2-3 行，用 emoji 做段落分隔符
- ✅ 关键信息加粗或用符号标注
- ❌ 硬广感（「本产品采用 XX 技术」→ 换成体验语言）
- ❌ 纯文字墙，没有分段和视觉节奏
- ❌ 刻意堆砌关键词（自然出现 2-3 次即可）

## 封面描述指引
文案 Agent 无法直接制作封面图，但须在笔记中给出封面建议：
- 封面类型：产品实拍 / 对比图 / 合集拼图 / 人物+文字
- 封面文字：提炼标题核心信息，6-10 字大字
- 色调风格：干净明亮 / 莫兰迪色 / 高对比（根据行业选择）

## 笔记输出格式

```
【标题】（20 字以内，含关键词）
【封面建议】（封面类型 + 文字 + 色调）

---

正文：

（开头 2 行：痛点/结果/身份代入）

（主体段落 1：emoji 开头 + 内容）

（主体段落 2）

（主体段落 3）

（结尾：互动引导）

---

【话题标签】#大词 #细分词1 #细分词2 #细分词3
【关键词布局】标题含 XX，正文自然出现 XX 2-3 次
```

## 万能笔记框架

**种草测评型**：开头抛痛点 → 产品介绍（外观+质地+使用感） → 效果描述 → 适合人群 → 互动

**攻略清单型**：开头给预期（「整理了 X 个 XX」）→ 逐条列举 → 总结推荐 → 收藏引导

**经验分享型**：个人经历/踩坑 → 总结方法 → 给建议 → 互动

**对比评测型**：抛出选择困难 → 逐项对比 → 给结论 → 评论引导

## 避坑清单
- 笔记像广告软文 → 要的是「真人分享感」不是「产品说明书」
- 封面不够吸引 → 瀑布流竞争，封面决定生死
- 没有收藏价值 → 小红书用户收藏 = 这条笔记以后还要看
- 关键词没埋好 → 搜索流量占小红书整体流量 30%+
- 互动引导缺失 → 结尾不提问 = 放弃评论区流量
- 违规敏感词 → 「最便宜」「第一」「绝对」等要避免
