# OCR / 表格识别 SOP v1

## 目标
把图片识别这件事从“凭感觉看图”改成有止损、有分级、有回退链路的流程。
重点不是提高玄学命中率，重点是：
- 少错
- 错了能及时停
- 关键数据不带毒进入自动化

## 一、任务分型
收到图片/截图后，先判断类型：

### A 类：普通文字图
- 海报
- 通知
- 对话截图
- 一段说明

风险：中低
处理：可先直接读，再标注置信度。

### B 类：结构化表格图
- 课表
- 清单
- 财务表
- 表单
- 配置表

风险：高
处理：优先结构化提取，不允许直接自由发挥下结论。

### C 类：关键决策图
- 要写进自动化规则
- 要进 cron
- 要进提醒系统
- 要做配置依据
- 要做长期记录

风险：最高
处理：必须二次确认。OCR 单源结果不能直接落地。

## 二、输入质量评估
直接判定为高风险输入的情况：
- 图片模糊
- 截图被压缩
- 裁切不完整
- 文字太小
- 表格格子密
- 一格里多行字
- 有遮挡
- 透视变形
- 同时存在多个相似字段（如周次 / 节次 / 时间）

一旦命中：
1. 标记为中低置信度
2. 明确说不适合作为自动化依据
3. 主动索要 CSV / Excel / PDF / 文字版 / 更清晰原图

## 三、识别流程
### Step 1：版型判断
先判断这是段落、表格、UI 页面、清单还是表单。
规则：先说版型，再开始识别。

### Step 2：字段抽取
如果是表格/清单类，必须先抽字段。
例如课表先抽：
- 课程名称
- 授课周次
- 任课教师
- 上课时间
- 上课地点

规则：先结构化，后解释。

### Step 3：冲突检查
检查是否存在：
- 周次互相矛盾
- 时间和节次不对应
- 同一时段两门课重叠但无解释
- 识别结果和图面明显不一致
- 同一字段前后两次读法不同

如果有冲突：
- 立即停止往下推理
- 标记低置信度
- 请求更可靠源
- 不进入自动化链路

### Step 4：置信度分级
#### 高置信度
- 图清晰
- 字段完整
- 无明显冲突
- 可用于整理/归档

#### 中置信度
- 大体能读
- 存在少量模糊字段
- 可做摘要
- 不可直接用于 cron / 配置 / 自动化

#### 低置信度
- 关键字段不清
- 多处冲突
- 结构复杂
- 只能粗看，必须换源

## 四、输出规则
### 普通图片
可以直接总结，但要带一句：
- 按图粗读
- 中等置信度
- 以下字段可能需要确认

### 表格图片
必须按顺序输出：
1. 结构化字段
2. 识别到的冲突 / 不确定点
3. 置信度
4. 是否建议继续用于自动化

规则：不要直接从模糊课表图推 cron，不要直接从截图配规则，不要直接把 OCR 结果当事实。

## 五、回退链路
当 OCR 不稳时，不允许原地打转，必须主动切备用链路。

优先级：
1. 原始结构化文件（CSV / Excel / JSON / 表格导出）
2. 文档文件（PDF / 可复制文本 / 原页面导出）
3. 更高清截图（不裁切 / 不压缩 / 正对全图）
4. 人工文字确认

## 六、自动化安全规则
任何 OCR 结果，只有同时满足以下条件才允许进入自动化：
1. 是高置信度
2. 关键字段已结构化
3. 没有冲突
4. 已由第二来源确认（文件源 / 用户文字确认 / 第二次识别一致）

否则：
- 不进 cron
- 不进提醒
- 不进配置
- 不写长期记忆为确定事实

## 七、主脑止损口径
### 低置信度
- 这张图我只能粗读，不适合直接拿来建规则。
- 如果你要让我做自动化，请给我 CSV / 文本版。

### 中置信度
- 我先帮你提字段，但其中几项还不稳，确认后再往下走。

### 高置信度
- 这版字段够稳，可以继续做整理 / 规则。

## 八、系统硬规则
1. 结构化表格图默认高风险。
2. 截图 OCR 不得直接作为自动化依据。
3. 关键字段必须先结构化抽取，再做推理。
4. 连续两次识别冲突时，立即换源。
5. 凡是进 cron / 配置 / 提醒的 OCR 结果，必须二次确认。
6. 优先级：原文件 > 文本 > 高清图 > 模糊截图。

## 一句话总结
OCR 能力要提升，不是先追求“看得更神”，而是先建立“什么时候不能信、什么时候必须换源”的纪律。
