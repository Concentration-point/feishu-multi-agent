# Skills Inventory

> 目的：给 Clawd 一份“能力台账”，避免重复推荐、重复学习、重复吹牛。

## 使用规则

- 先查这里，再谈“学新 skill”或“推荐 skill”。
- 命中以下任一类，就不要再当作“新增候选”推荐：
  - 已安装且已掌握
  - 已安装且已配置过类似能力
  - 已明确指定为默认工作流的一部分
- 只有“未安装 / 未学透 / 当前确实缺能力”的项，才算新增候选。

## D. 新增 skill 推荐前检查清单

推荐前必须先回答这 5 个问题：

1. 这个 skill 是不是已经安装？
2. 我是不是已经学过、用过、或被老大明确配置过类似能力？
3. 它解决的是不是当前真实短板，而不是已有能力的变体？
4. 风险是否讲清楚：权限、数据访问、联网、写入、外发、持久化、供应链、回滚？
5. 老大有没有明确说“继续 / 执行 / 可以学”？

只要有一项没过，就不能把它当成正式推荐。

## F. 与主动规则的关系

- 本文件是所有 skill 相关“主动推荐/主动学习提案”的前置检查表。
- 若 `config/proactive-rules.md` 鼓励主动建议新能力，也必须先经过本文件排除已学/已装/已配置项。
- 本文件优先阻止“重复推荐”，再允许“新增提案”。

## G. 以后更新规则

发生以下情况时更新本文件：
- 安装了新 skill
- 明确学会了某个 skill
- 老大指出“这个你已经学过/配过”
- 某个 skill 成为默认工作流的一部分
- 某个外部项目的方法论已被正式吸收到本地 SOP
