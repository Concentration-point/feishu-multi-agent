# Output Separation Rules

目标：内部调试和用户可见输出彻底分层。

## 用户可见输出只允许包含
- 结论
- 当前状态
- 关键风险
- 下一步

## 不直接输出给用户的内容
- 原始 HTML
- 栈追踪
- 原始工具日志
- 编码修复细节
- 中间调试术语
- 未清洗的 OCR 中间结果

## 自动化失败时固定口径
1. 哪一步失败
2. 当前结果是否已送达
3. 下一步怎么办

## 晨报/固定播报类额外规则
- 不暴露抓取链细节
- 不暴露编码修复过程
- 不把内部异常术语混入成品

## 长任务规则
- 调试信息进文件或任务状态
- 对话里只保留摘要和决定
