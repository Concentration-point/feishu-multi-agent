# Automation Rules

> 目的：把晨报、晚报、提醒、定时任务这些自动化活做得像系统，不像抽奖。

## 1. 通用原则

- 自动化任务的输出必须**固定格式**，不要每次风格漂移。
- 能自动投递到当前会话的任务，不要再额外承诺“我稍后手动发你”。
- 如果任务失败、超时、中断，必须明确报：
  - 哪一步失败
  - 当前结果是否已送达
  - 下一步怎么办
- 自动化输出优先：**短、稳、可验证**。
- 一旦用户明确了唯一模板，后续必须以该模板为准，不得混搭旧版规则。

## 2. 晨报规则

### 2.1 唯一固定结构

晨报固定按以下顺序输出，并且不得增删、换序、混搭：

1. 南京天气（固定开头）
2. 头条（一句话 + 一句锐评）
3. 国内（一句话 + 一句锐评）
4. 国际（一句话 + 一句锐评）
5. 科技 / AI（一句话 + 一句锐评）
6. 行业（一句话 + 一句锐评）
7. 知乎热榜（带高赞观点）
8. 微博热搜（人话版）
9. 今日结论
10. 每日英语（固定结尾，带出处）

要求：
- 南京天气必须是开头第一栏。
- 每日英语必须是结尾最后一栏。
- 国内 / 国际 / 科技AI / 行业 这四栏都按“一句话 + 一句锐评”写，不再使用旧版的“发生了什么 / 为什么值得看”模板。
- 整体必须是人话版，不是标题墙，不是公文，不是资讯 App 复制。
- 如果某栏缺少可靠信息，明确写：`暂无可靠增量`。

### 2.2 排版规则（飞书清爽版）

- 禁止大量使用 `*`、`**`、`***` 做整段强调。
- 只允许在**极少数关键词**上加粗；如果不用加粗也能看懂，那就别加。
- 优先使用：
  - 空行分块
  - 简洁栏目标题
  - `1.` `2.` 这种自然编号
  - `-` 这种轻量列表
- 不要把每一句都包成强调文本。
- 不要用花哨符号堆版面。
- 整体观感目标：像一份干净的编辑简报，不像没渲染好的 Markdown。

推荐样式示例：

早报｜2026-03-17

南京天气
天气：多云转小雨
温度：10-18°C
出门建议：带伞，早晚偏凉

头条
1. 一句话：...
   锐评：...
   来源：...

国内
1. 一句话：...
   锐评：...
   来源：...

不要写成这种：
- `**头条**`
- `**一句话：...**`
- `**锐评：...**`
- 全篇满屏星号强调

### 2.3 各栏写法

- **南京天气**
  - 固定写法建议：
    - `天气：...`
    - `温度：...`
    - `出门建议：...`
  - 一眼说清今天怎么穿、要不要带伞。

- **头条 / 国内 / 国际 / 科技AI / 行业**
  - 每栏优先 1-3 条，少而精。
  - 每条固定写法：
    - `一句话：...`
    - `锐评：...`
    - `来源：...`
  - `一句话` 负责概括事件。
  - `锐评` 必须有判断，有观点，别温吞。
  - 不准写成长段废话。

- **知乎热榜**
  - 选 2-3 条真正值得看的。
  - 每条必须包含：
    - `问题：...`
    - `高赞观点：...`
    - `为什么值得看：...`
    - `来源：知乎热榜 / 回答者`
  - 不能只报题目，必须把高赞回答里的真实观点带出来。

- **微博热搜**
  - 选 3-5 条。
  - 每条必须包含：
    - `词条：...`
    - `人话版：...`
    - `来源：微博热搜`
  - 重点是把词条翻成人能读懂的话。
  - 不要为了凑数塞低信息量娱乐词条，除非它真有公共讨论价值。

- **今日结论**
  - 用 2-4 句收尾。
  - 先说今天最值得盯的一件事，再说它对老大的实际意义。

- **每日英语**
  - 必须作为全文最后一个栏目。
  - 必须包含：
    - 英文原句
    - 中文翻译
    - 具体出处
  - 出处格式固定：
    - `出处：书名 / 作者 / 演讲 / 采访 / 电影 / 文章标题`
  - 禁止来源不清的鸡汤句。

### 2.4 每日英语去重规则

- 读取 `memory/daily-english-history.md`
- 检查最近 **14 天**
- 重复或高度相似则必须换一句
- 发送后追加：`YYYY-MM-DD | English | 中文 | 出处`

## 3. 晚报规则

- 晚报强调：**今天世界发生了什么，5 分钟读完**。
- 每栏只保留真正有增量的信息。
- 没有可靠增量就写：`今天无显著新增`
- 避免把晨报内容原样复读一遍。
- 晚报只保留 OpenClaw 具体实施场景 / 实操案例 / 工作流 / 踩坑总结，其他内容全部取消。
- 晚报检索源优先 GitHub 与官方相关来源；没有新增且没价值时，可以不发。

## 4. Reminder / Cron 规则

- Reminder 文案必须像提醒，而不是系统消息。
- 若为自动投递型 cron：
  - 正确说法：`已触发，结果会由系统自动发到当前对话。`
  - 错误说法：`跑完我会发给你。`
- 不要制造错误预期。

## 5. 进度反馈规则

- 不发悬空进度回复。
- 只有两种情况允许先发进度：
  1. 同一轮马上继续并给结果
  2. 明确后台执行，并会有自动结果 / 明确后续通知
- 否则直接给结果，不要先嘴上答应。

## 6. 与主动规则的关系

- 自动化任务允许“主动补强”，但不允许“主动改制”。
- 若 `config/proactive-rules.md` 鼓励主动建议，而自动化模板要求固定结构，则自动化模板优先。
- 在自动化场景中，主动性只应用于：
  - 去重
  - 来源补全
  - 错误兜底
  - 内容压缩
- 不应用于：
  - 擅自改栏目
  - 擅自扩写
  - 擅自改变交付方式

## 7. 后续维护

发生以下情况时更新本文件：
- 晨报/晚报格式再次跑偏
- 自动化输出出现重复内容
- 定时任务交付方式让人误解
- 新增了新的固定自动化任务
