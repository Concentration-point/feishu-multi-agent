# 抖音通用规则

## 平台核心逻辑

抖音产出物是**口播脚本或拍摄脚本**，不是图文。算法权重：完播率 > 互动率（评论 > 转发 > 点赞）> 关注转化率。

## 时长控制
- 引流内容：15-30 秒
- 种草/探店：30-60 秒
- 深度测评/剧情：60-120 秒
- 口播字数 ≈ 时长（秒）× 4-5 字

## 三段式结构（所有行业通用骨架）

| 段落 | 时间占比 | 任务 |
|------|---------|------|
| 开头·钩子 | 前 3 秒 | 让用户停下来 |
| 中段·价值 | 60-70% | 交付信息或情绪，一条只说一件事 |
| 结尾·转化 | 最后 3-5 秒 | 引导行动 |

## 开头钩子公式（任选其一）
1. 点名式：「在 XX 的朋友注意了」
2. 冲突式：「千万别再 XX 了」
3. 利益式：「教你一招省 XX 元」
4. 悬念式：「你知道 XX 为什么 XX 吗？」
5. 数字式：「3 个技巧」「5 秒搞定」
6. 反常识：「XX 行业 99% 的人都不知道」

## 结尾转化公式（任选其一）
1. 评论引导：「你觉得哪个更好？评论区告诉我」
2. 关注引导：「关注我，下期教你 XX」
3. 到店引导：「就在 XX 路，周末约起来」
4. 购买引导：「链接放在左下角，手慢无」
5. 收藏引导：「先收藏，用的时候找得到」

## 文案风格
- ✅ 口语化、短句、有节奏感、像跟朋友聊天
- ✅ 用「你」不用「您」，用「咱」拉近距离
- ❌ 长句、书面语、公众号腔
- ❌ 一条内容塞太多信息
- ❌ 开头铺垫超过 5 秒

## 脚本输出格式

```
【标题】视频标题/话题标签建议
【时长】XX 秒
【类型】口播 / 探店实拍 / 产品展示 / 剧情 / 教程演示

---

[0-3s] 开头·钩子
画面：（场景/动作）
口播：「……」

[3-XXs] 中段·主体
画面：（镜头/展示）
口播：「……」

[XX-XXs] 结尾·转化
画面：（收尾）
口播：「……」

---

【BGM 建议】节奏类型
【拍摄提示】关键要点
```

## 万能脚本框架

**行业揭秘型**：钩子抛行业潜规则 → 指出用户不是不聪明而是被坑 → 给正确做法

**省钱/省时型**：钩子抛惊人效果 → 1-2-3 步骤演示 → 收藏引导

**对比型**：钩子抛选择困难 → 逐项对比 → 给建议 + 评论引导

**故事型**：设置冲突悬念 → 故事发展中植入产品 → 反转 + 行动引导

## 避坑清单
- 文案像公众号 → 要的是说话感不是阅读感
- 开头没钩子 → 前 3 秒没抓力 = 秒划
- 信息过散 → 一条只说一件事
- 没转化引导 → 看完不知道干什么 = 浪费流量
- 忽略字幕 → 很多用户静音刷，字幕是第二条生命线
- 违规词 → 绝对化用语、功效承诺要警惕
