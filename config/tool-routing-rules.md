# Tool Routing Rules

目标：把工具切换从临场发挥，变成默认纪律。

## 1. 网页检索
- `web_fetch` 失败一次：改 `web_search`
- 仍失败：改浏览器/渲染链
- 再失败：明确降级，不死撞

## 2. 热榜/晨报链
- 官方页失败：改聚合页
- 聚合页脏数据：改搜索恢复版
- 搜索仍不稳：输出人话替代版，并明确标注

## 3. OCR / 图片识别
- 结构化图先抽字段，后解释
- 关键字段不清：停止自动化，不硬猜
- 连续两次冲突：强制换源

## 4. Coding
- 多文件 / 重构 / 长任务：默认走 `coding-agent`
- 小修小补：可直接修改

## 5. 自动化实验
- 遇到 Cloudflare / challenge：先分析入口与链路，不重复裸点同一路线超过 2 次
- 页面问题优先区分：入口错误 / 会话问题 / 页面结构变化

## 6. 记账图片
- 金额 / 商户 / 状态任一不清：不入账
- 用户主动发送金额相关内容：默认按已确认消费处理
- 一旦对外使用“可直接记 / 已确认消费 / 已记 / 已入账”等确定性表述，必须把“正式写账本 + 回读核对”当成同一动作；若做不到，宁可回复“待补记/未落账”，不准先说记上了
