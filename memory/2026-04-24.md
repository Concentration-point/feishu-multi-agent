# 2026-04-24

- 今天首次把“微信文章阅读失败”的根因打实：不是不会读网页，而是默认没有可复用的真实浏览器/CDP 接管链路。
- 这次已验证的最低风险本机接管方案：本机启动独立 Chrome + `--remote-debugging-port=9222` + 独立 `--user-data-dir=$env:TEMP\openclaw-chrome-cdp`，再用 `agent-browser --cdp 9222` 接管。
- 已验证可行流程：启动 Chrome -> `Invoke-WebRequest http://127.0.0.1:9222/json/version` 验证 -> `agent-browser --cdp 9222 open <url>` -> `wait` -> `eval \"document.body.innerText\"` 抽正文。
- 教训：以后凡是用户说“浏览器打开/接管/阅读登录态页面”，不要先嘴上答应“能接管”；必须先验证 CDP 入口，再执行。
- 老大明确要求：把这次浏览器接管好好纳入 SOP，避免每次都不会。
