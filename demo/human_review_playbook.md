# Demo 人机协同演练脚本

## 概述

本文档为 Demo 现场"人机协同审核"环节的操作指南。客户经理 Agent 生成 Brief 解读后，会通过飞书群聊发送审核卡片，等待人类审核员回复。

**审核员角色**：需要一位队友全程在飞书群聊中待命，在看到橙色审核卡片后 15-30 秒内回复。

---

## 环境准备

### 1. 确认配置

```bash
# .env 中确保以下配置正确
FEISHU_CHAT_ID=oc_xxxxxxxxx       # 群聊 ID
AUTO_APPROVE_HUMAN_REVIEW=false   # Demo 时设为 false，手动审核
HUMAN_REVIEW_TIMEOUT=300          # 超时秒数，Demo 建议 120
HUMAN_REVIEW_POLL_INTERVAL=3      # 轮询间隔，Demo 建议 3 秒
```

### 2. 快速跑通模式（彩排用）

```bash
# 如果只是跑通流程不需要人工介入
AUTO_APPROVE_HUMAN_REVIEW=true
```

### 3. 备用跳过方案

如果现场网络故障或 IM 异常：
1. 立即修改 `.env` 中 `AUTO_APPROVE_HUMAN_REVIEW=true`
2. 重启流程（或等待超时自动通过）
3. 向观众说明："为保证 Demo 流畅，此处切换为自动审批模式"

## 三个 Brief 的预设回复话术

### Brief A：电商大促（某护肤品牌）

**场景**：双十一玻尿酸精华液大促

**预设回复（需要修改）**：
```
修改：客户说的"25-35岁女性"实际核心是 25-30 岁精致白领，不是泛年龄段。"科技感"指的是成分实证风格（如临床数据、专利成分），不是赛博朋克视觉。另外客户口头提过双十一预热期从 10 月 20 日开始，这个时间窗口要写进解读。
```

**说明**：这条回复会触发客户经理 Agent 修改 Brief 解读，然后再次提交审核。第二次可以回复"通过"。

**第二轮回复**：
```
通过
```

### Brief B：新品发布（某母婴品牌）

**场景**：婴幼儿分龄营养包上市

**预设回复（直接通过）**：
```
通过
```

**说明**：母婴品牌 Brief 信息相对完整，直接通过可以展示"不同场景不同审核结果"的能力。

### Brief C：品牌传播（某新消费品牌）

**场景**：品牌升级传播

**预设回复（需要修改）**：
```
修改：客户说的"年轻白领"具体指一线城市 25-32 岁、月收入 1.5w+ 的消费群体。品牌升级的核心是从"性价比"心智转向"品质生活"心智，这个转型方向必须写进解读。还有客户明确说过不要用"国潮"这个词。
```

**第二轮回复**：
```
通过
```

## 操作流程

### 正式 Demo 流程

```
1. 主讲人启动 Pipeline
   $ python demo/run_demo.py --scene 电商大促

2. 客户经理 Agent 开始工作
   → 读取 Brief → 搜索知识库 → 生成初版解读

3. 群聊出现橙色卡片 "🔍 Brief 解读等待审核"
   → 审核员打开飞书群聊

4. 审核员在 15-30 秒内发送预设回复
   → 粘贴对应 Brief 的预设话术

5. Agent 收到反馈，修改解读
   → 如果是"修改"，Agent 会重新生成并再次发卡片

6. 第二轮审核，审核员回复"通过"

7. 客户经理完成，流程进入策略师阶段
```

### 时间预估

| 环节 | 耗时 |
|------|------|
| Agent 生成初版解读 | 15-30s |
| 发送审核卡片 + 人类回复 | 15-30s |
| Agent 修改解读 | 10-20s |
| 第二轮审核 + 人类回复 | 10-15s |
| **客户经理阶段总计** | **50-95s** |

### 审核员注意事项

1. **速度**：看到卡片后尽快回复，保持 Demo 节奏
2. **格式**：修改意见必须以"修改："开头，否则 Agent 可能误判
3. **手机备用**：在电脑和手机上都打开飞书群聊，防止一端卡顿
4. **不要多发**：一轮审核只发一条消息，多条可能导致解析混乱

## 答辩解说词参考

### 当审核卡片出现时

> "大家注意看群聊——客户经理 Agent 刚刚生成了 Brief 解读，它没有直接往下推进，而是主动发到群聊请人类专家审核。这就是我们的'人机协同'设计：AI 不是自说自话，关键决策点有人类把关。"

### 当人类给出修改意见后

> "我们的同事刚刚给出了修改意见，指出了 Agent 对目标人群的理解偏差。大家看——Agent 正在根据这个反馈重新生成解读。更关键的是，这条'人类修正'会被蒸馏成经验写入知识库。下次遇到类似客户，Agent 一开始就能解读对。"

### 当第二轮通过时

> "通过了。从收到 Brief 到人类审核完毕，总共不到一分钟。传统流程这个环节至少要几个小时的邮件往返。"

## 故障处理

| 问题 | 处理方式 |
|------|---------|
| 审核卡片没出现 | 检查 FEISHU_CHAT_ID 配置，看终端日志 |
| Agent 没收到回复 | 检查机器人是否有 `im:message:readonly` 权限 |
| 回复了但 Agent 无反应 | 确认回复是在群聊中发的（不是私聊），检查 poll interval |
| 超时自动通过了 | 说明回复太慢或消息未被识别，下次注意格式 |
| 现场网络故障 | 立即改 `AUTO_APPROVE_HUMAN_REVIEW=true`，重启 |
