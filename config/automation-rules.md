# Automation Rules

> 目的：把晨报、晚报、提醒、定时任务这些自动化活做得像系统，不像抽奖。

## 1. 通用原则

- 自动化任务的输出必须**固定格式**，不要每次风格漂移。
- 能自动投递到当前会话的任务，不要再额外承诺“我稍后手动发你”。
- 如果任务失败、超时、中断，必须明确报：
  - 哪一步失败
  - 当前结果是否已送达
  - 下一步怎么办
- 自动化输出优先：**短、稳、可验证**。

## 2. 晨报规则

### 2.1 固定结构

1. 南京天气
2. 热点
3. OpenClaw
4. ClawHub / Skill 观察
5. 每日英语

### 2.2 热点规则

- 热点必须是**人话版**，不是标题改写。
- 每条固定格式：
  - `发生了什么：...`
  - `为什么值得看：...`
  - `来源：媒体/机构名`
- 每条控制在 **2 句内**。
- 禁止：
  - 标题拼盘
  - 套话
  - 官腔
  - 无来源信息

### 2.3 每日英语规则

- 必须包含：
  - 英文原句
  - 中文翻译
  - 具体出处
- 出处格式固定：
  - `出处：书名 / 作者 / 演讲 / 采访 / 电影 / 文章标题`
- 禁止来源不清的鸡汤句。
- 去重规则：
  - 读取 `memory/daily-english-history.md`
  - 检查最近 **14 天**
  - 重复或高度相似则必须换一句
  - 发送后追加：`YYYY-MM-DD | English | 中文 | 出处`

## 3. 晚报规则

- 晚报强调：**今天世界发生了什么，5 分钟读完**
- 每栏只保留真正有增量的信息。
- 没有可靠增量就写：`今天无显著新增`
- 避免把晨报内容原样复读一遍。

## 4. Reminder / Cron 规则

- Reminder 文案必须像提醒，而不是系统消息。
- 若为自动投递型 cron：
  - 正确说法：`已触发，结果会由系统自动发到当前对话。`
  - 错误说法：`跑完我会发给你。`
- 不要制造错误预期。

## 5. 进度反馈规则

- 不发悬空进度回复。
- 只有两种情况允许先发进度：
  1. 同一轮马上继续并给结果
  2. 明确后台执行，并会有自动结果 / 明确后续通知
- 否则直接给结果，不要先嘴上答应。

## 6. 与主动规则的关系

- 自动化任务允许“主动补强”，但不允许“主动改制”。
- 若 `config/proactive-rules.md` 鼓励主动建议，而自动化模板要求固定结构，则**自动化模板优先**。
- 在自动化场景中，主动性只应用于：
  - 去重
  - 来源补全
  - 错误兜底
  - 内容压缩
- 不应用于：
  - 擅自改栏目
  - 擅自扩写
  - 擅自改变交付方式

## 7. 后续维护

发生以下情况时更新本文件：
- 晨报/晚报格式再次跑偏
- 自动化输出出现重复内容
- 定时任务交付方式让人误解
- 新增了新的固定自动化任务
